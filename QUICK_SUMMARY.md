# Jelsa.by - Краткое резюме оптимизации

## ✅ Что исправлено (2026-04-14)

### 1. Главная проблема: медленная сборка (5 минут каждый раз)

**Решение**: Создан `src/js/webp-optimized.mjs` с MD5-кэшированием

**Результат**:
- Первая сборка: ~5-10 минут (обрабатывает все 1595 изображений)
- Последующие сборки: **10-30 секунд** (только измененные файлы)
- **Ускорение в 10-30 раз!**

**Команды**:
```bash
npm run build        # Быстрая сборка (инкрементальная)
npm run build:full   # Полная пересборка всех изображений
```

**Как работает**:
- Создается `.webp-cache.json` с MD5-хэшами файлов
- При повторной сборке конвертируются только новые/измененные изображения
- Кэш-файл добавлен в `.gitignore`

### 2. Альтернатива без jQuery

**Файл**: `src/js/index-no-jquery.js`

**Экономия**: ~30KB в бандле

**Чтобы использовать**:
1. В `webpack.config.js` замени:
```js
main: {
  import: "./js/index-no-jquery.js",  // было: "./js/index.js"
},
```
2. Удали jQuery из dependencies
3. Повтори для других файлов: `gallery.js`, `expanded.js`, `faq.js`, `bar-list.js`

## 🔴 Критичные проблемы (требуют исправления)

### 1. Огромные изображения (3.6-5.2 MB)
- **Проблема**: Файлы в `party/` весят по 3-5 мегабайта
- **Решение**: Использовать sharp для лучшего сжатия + resize
- **Ожидаемый результат**: 5MB → 300-500KB

### 2. Google Analytics не настроен
- **Файл**: `src/index.html:32,38`
- **Проблема**: Плейсхолдер `GA_TRACKING_ID` не заменен
- **Решение**: Заменить на реальный ID или удалить код

### 3. Three.js загружается на всех страницах
- **Файл**: `src/index.html:52-53`
- **Проблема**: Three.js нужен только для `/view-tour-1.html`
- **Решение**: Переместить скрипты в `view-tour-1.html`

### 4. Console.log в продакшене
- **Файл**: `src/js/expanded.js:128`
- **Решение**: Удалить `console.log(textArray[i]);`

### 5. Кэширование отключено
- **Файл**: `src/index.html:26-28`
- **Проблема**: Мета-теги отключают кэш статики
- **Решение**: Удалить эти строки
